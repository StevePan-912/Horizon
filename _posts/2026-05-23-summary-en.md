## [Memory shortage causes repricing of consumer electronics](https://simonwillison.net/2026/May/22/memory-shortage/#atom-everything) ⭐️ 8.0/10

AI-driven demand for HBM memory is consuming increasing wafer capacity from the three major memory manufacturers, causing shortages and price increases for consumer electronics that use DDR and LPDDR memory. HBM's share of wafer allocation has grown from 2% to an expected 20% by end of 2026. This represents a fundamental shift in semiconductor supply chains where AI infrastructure demand is restructuring consumer electronics pricing through memory shortages. The impact is particularly severe in emerging markets like Africa and South Asia where sub-$100 smartphones are affected. A single gigabyte of HBM consumes more than three times the wafer capacity that a gigabyte of DDR or LPDDR does, creating disproportionate pressure on manufacturing capacity. Memory companies prioritize high-margin HBM production over consumer RAM due to profit considerations.

rss · Simon Willison · May 22, 22:01

**Background**: HBM (High Bandwidth Memory) is a 3D-stacked DRAM technology that provides unprecedented memory bandwidth by vertically integrating multiple DRAM dies, making it essential for AI accelerators. DDR is used in desktops and servers for performance, while LPDDR is designed for low-power consumption in mobile devices. The semiconductor industry has only three major memory manufacturers with fixed wafer processing capacity that must be allocated between different memory types.

## [Why Japanese companies pursue broad diversification strategies](https://davidoks.blog/p/why-japanese-companies-do-so-many) ⭐️ 6.0/10

Analysis reveals that Japanese companies pursue broad diversification through lifetime employment systems and employee-focused governance rather than shareholder optimization, creating companies that exist primarily to continue existing. This represents a fundamentally different corporate model from Western focus strategies, offering insights into alternative business approaches that prioritize employee stability and long-term continuity over short-term profits. The system relies on 'shūshin koyō' (lifetime employment) where employees develop company-specific skills rather than transferable occupational skills, making the company insulated from outside pressure and focused on employee welfare rather than shareholder returns.

hackernews · d0ks · May 22, 15:22 · [Discussion](https://news.ycombinator.com/item?id=48237163)

**Background**: Japanese companies historically adopted 'shūshin koyō' (lifetime employment) starting in the 1920s, which became a defining characteristic of Japanese corporate culture. This system involves hiring employees fresh out of school with the expectation they will stay until retirement, developing specialized skills tailored to the specific company rather than general marketable skills. Unlike Western companies that prioritize shareholder value and focus on core competencies, Japanese firms operate with employee-centered governance structures that emerged after World War II as part of Japan's economic reconstruction efforts.

## [Deno 2.8 Release Announced](https://deno.com/blog/v2.8) ⭐️ 6.0/10

Deno 2.8 has been released with new features including the deno pack command for safe and simple packaging of applications. The release includes various improvements to the JavaScript and TypeScript runtime environment. This release continues Deno's evolution as a modern JavaScript runtime with strong security defaults and TypeScript integration. The comparison with Node.js and Bun highlights ongoing competition in the JavaScript runtime space, affecting developer choices for new projects. Deno features a permission-based security model that requires explicit grants for file, network, and other system access. The new deno pack command provides single-executable output capabilities similar to Node.js's upcoming features.

hackernews · roflcopter69 · May 22, 11:23 · [Discussion](https://news.ycombinator.com/item?id=48234380)

**Background**: Deno is a JavaScript and TypeScript runtime created by Ryan Dahl, the original creator of Node.js. It addresses some of Node.js's design limitations with features like built-in TypeScript support, secure defaults through permission system, and a standard library. Unlike Node.js which uses the V8 engine, Deno also uses V8 but with different architectural decisions focused on security and simplicity. Bun is another JavaScript runtime that competes with both Node.js and Deno, using JavaScriptCore instead of V8 and offering an all-in-one toolkit approach.
