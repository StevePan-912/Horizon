## [Developer reverse engineers Apple's video wallpaper system](https://github.com/kageroumado/phosphene) ⭐️ 7.0/10

A developer created Phosphene, a tool that reverse engineers Apple's WallpaperExtensionKit.framework to enable custom video wallpapers on macOS desktops. The tool allows users to add their own videos as desktop wallpapers, which macOS doesn't natively support. This provides users with enhanced customization options for macOS desktops, allowing them to use personal videos as wallpapers instead of being limited to Apple's pre-installed options. The technical achievement demonstrates how reverse engineering can unlock functionality that Apple hasn't made available to users. The tool uses AVSampleBufferDisplayLayer to render videos directly with PTS-offset gapless looping, and includes intelligent features that pause or downshift playback based on thermal state, battery level, brightness, and window occlusion. Unlike Apple's Aerials which only play on the lock screen, these custom videos continue playing on the desktop.

hackernews · kageroumado · May 20, 23:54 · [Discussion](https://news.ycombinator.com/item?id=48215979)

**Background**: Apple's video wallpapers were introduced as a premium feature in macOS, but they're limited to Apple's own content like the Aerial videos of nature scenes. WallpaperExtensionKit.framework is Apple's private framework that manages wallpaper functionality in System Settings. AVSampleBufferDisplayLayer is an Apple framework component that handles video rendering by decompressing and displaying video frames directly. PTS (Presentation Timestamp) offsetting is a technique used to maintain synchronization between audio and video streams during playback.

## [Saying goodbye to asm.js](https://spidermonkey.dev/blog/2026/05/20/saying-goodbye-to-asmjs.html) ⭐️ 7.0/10

Mozilla announces the end of asm.js support, marking the complete transition from asm.js to WebAssembly for high-performance browser applications. This represents a major milestone in web platform evolution, showing how WebAssembly has successfully replaced asm.js as the standard for running native-speed code in browsers. The shift affects how developers compile and deploy performance-critical applications like games and design tools. asm.js was a strict subset of JavaScript that allowed C/C++ code to run at near-native speeds in browsers, but required parsing JavaScript ASTs and had larger bundle sizes compared to WebAssembly. WebAssembly provides better load times and performance through its binary format.

hackernews · eqrion · May 20, 12:01 · [Discussion](https://news.ycombinator.com/item?id=48206340)

**Background**: asm.js was Mozilla's response to running native-speed code on the web, created as a subset of JavaScript that could be optimized for performance by restricting language features. It served as a precursor to WebAssembly, which was announced in 2015 and became a W3C recommendation in 2019. Companies like Figma used asm.js to prove that complex desktop applications could run in browsers before migrating to WebAssembly.
